# 媒体设置

**媒体** 页集中管理 AI 重命名、STRM、CAS 秒传、Emby、CloudSaver、Alist、TMDB、媒体库分类，以及影巢的 **Cookie 兜底** 配置。

保存走 `POST /api/settings/media`。影巢的 Bridge、自动签到、OpenAPI 等完整项在 **系统** 页，见 [[SystemSettings]] / [[HDHive]]。

---

## 1. AI 辅助重命名

AI 辅助重命名用于整理器、任务重命名和媒体识别。

| 配置项 | 说明 |
| :--- | :--- |
| 启用 / 停用 AI | 控制 AI 功能是否生效 |
| API 地址 | OpenAI 兼容接口地址 |
| API Key | 接口密钥 |
| 模型名称 | 使用的模型名称 |
| 剧集命名模板 | 剧集整理后的文件名模板 |
| 电影命名模板 | 电影整理后的文件名模板 |
| AI API 流控 | 限制 AI 请求频率 |

模板示例：

```text
电视剧：{name} - {se}{ext}
电影：{name} ({year}){ext}
```

常用变量包括 `{name}`、`{season}`、`{episode}`、`{se}`、`{year}`、`{ext}`，实际支持情况以当前版本为准。

## 2. STRM 设置

| 配置项 | 说明 |
| :--- | :--- |
| 启用 STRM 生成 | 普通任务完成后生成 STRM（`strm.enable`） |
| 普通任务使用系统中转 | 普通任务的 STRM 使用本项目代理地址（`strm.useStreamProxy`） |

更完整的配置方式请参考 [[StrmGuide]]。

---

## 3. 秒传设置

| 配置项 | 说明 |
| :--- | :--- |
| 启用家庭中转 | 秒传时通过家庭云中转，规避个人云风控 |
| 优先使用家庭中转 | 默认先尝试家庭云，失败后回退个人云 |
| 恢复后删除 CAS 文件 | 秒传恢复成功后删除 `.cas` 存根文件 |
| 生成后删除源文件 | 生成 CAS 后自动删除源文件 |

自动恢复路径监控、扫描间隔等见 **秒传** 页与 [[CAS]]（`/api/cas/auto-restart-config`）。

---

## 4. Emby 设置

| 配置项 | 说明 |
| :--- | :--- |
| Emby 地址 | Emby/Jellyfin 服务地址 |
| API Key | 媒体服务器 API Key |
| 启用 Emby 入库通知 | 任务完成后通知媒体服务器刷新 |
| 启用 Emby 反代播放 | 使用本项目反代播放地址 |
| 代理端口 | 独立代理端口，默认 `8097` |
| 启用下一集预热 | 监听播放会话并预热下一集 |
| Sessions 轮询间隔(ms) | 检查播放会话的间隔 |
| 预热去重时长(ms) | 避免短时间重复预热同一内容 |

如果使用 Docker，记得映射代理端口，例如 `-p 8097:8097`。详见 [[Emby]]。

---

## 5. CloudSaver 设置

CloudSaver 用于资源搜索、自动追剧候选资源检索，以及任务失败后的自动换源。

| 配置项 | 说明 |
| :--- | :--- |
| 服务地址 | CloudSaver 服务地址 |
| 用户名 | CloudSaver 登录用户名 |
| 密码 | CloudSaver 登录密码 |

推荐使用项目 README 中提到的修改版 CloudSaver。保存后会清理旧的 CloudSaver 登录缓存。

## 6. 影巢设置（媒体页 · 兜底）

媒体页影巢区块仅包含：

| 配置项 | 说明 |
| :--- | :--- |
| 启用影巢 | 总开关 |
| 站点地址 | 默认 `https://hdhive.com` |
| 网页登录 Cookie | Cookie 兜底解析 |

**Browser Bridge、网页登录账号密码、自动签到、OpenAPI 凭证** 请到 **系统** 页配置。详见 [[HDHive]]。

## 7. Alist 设置

| 配置项 | 说明 |
| :--- | :--- |
| Alist 地址 | Alist 服务地址 |
| Token | Alist 管理员或可用 API Token |

系统会检查 `alist.enable`、`alist.baseUrl`、`alist.apiKey` 均有效时才启用。详见 [[Alist]]。

## 8. TMDB 刮削

| 配置项 | 说明 |
| :--- | :--- |
| 启用刮削 | `tmdb.enableScraper`，总开关；关闭后相关刮削流程不生效 |
| TMDB API Key | 搜索剧集/电影、获取标题、年份、海报和集数等 |

TMDB 配置会影响自动追剧、任务刮削、整理器识别、海报墙等功能的准确度。

## 10. 正则预设管理

媒体页提供 **正则预设管理**，用于维护任务创建时可复用的过滤和重命名规则。字段与 [[Regex]] 一致。

---

## 11. 保存配置

修改媒体页配置后：

- 点击 **保存媒体设置** 保存。
- 点击 **放弃修改** 恢复到保存前状态。

保存 CloudSaver 配置后，系统会清理旧的 CloudSaver 登录缓存；保存 Emby/CAS 等配置后，相关服务会按需要重新加载。

---

## 配置建议

- 自动追剧用户优先配置 TMDB（含启用刮削）和 CloudSaver。
- 媒体服务器用户优先配置 STRM、Emby 和代理端口。
- 影巢完整能力请先配系统页 Bridge，再回影巢页验证签到/解锁。
- 文件名混乱时再开启 AI，简单过滤优先使用正则预设。
