# 任务管理

任务是 Cloud189 Auto Save 的核心执行单元。无论来源是分享链接、CloudSaver、自动追剧、订阅还是影巢资源，最终都会转成任务，由任务系统负责转存、更新检查、刮削、整理、STRM 和通知。

---

## 1. 常用操作

任务页支持以下操作：

- **创建任务**：填写分享链接、访问码、目标目录和任务配置。
- **批量创建**：一行一个分享链接，适合一次导入多个资源。
- **执行任务**：立即检查并转存指定任务。
- **执行全部**：按任务状态和调度规则批量执行。
- **编辑任务**：调整保存目录、集数、分组、备注、Cron、刮削、懒 STRM、整理器等配置。
- **删除任务**：删除任务记录；是否删除关联文件取决于对应操作入口。

---

## 2. 清理任务缓存

任务菜单新增 **清理缓存** 操作，用于删除指定任务的转存记录缓存。

适用场景：

- 分享目录内容变更，但系统仍按旧缓存判断。
- 手动删除或移动过网盘文件，需要重新检查。
- 任务异常失败后，希望重新计算已转存文件。

清理缓存不会删除云盘文件，只会让后续执行重新检查分享目录和目标目录。

---

## 3. 手动绑定 TMDB

任务菜单新增 **绑定 TMDB** 操作，用于修正自动识别不准确的媒体信息。

需要填写：

| 字段 | 说明 |
| :--- | :--- |
| TMDB ID | 电影或剧集在 TMDB 中的 ID |
| 媒体类型 | `movie` 或 `tv` |
| 手动季度 | 剧集可填季度号；留空则从任务名推断 |
| 显示标题 | 可选，留空使用 TMDB 返回标题 |

绑定成功后，任务会保存 `tmdbId`、媒体类型、标题和季度信息。剧集任务会尝试把同一资源组内未绑定的兄弟任务一起级联绑定，避免每一季重复手动配置。

---

## 4. 与整理器和 STRM 的关系

手动 TMDB 绑定会影响后续媒体联动：

- **刮削**：优先使用手动绑定的 TMDB ID 和季度。
- **整理器**：可使用更准确的标题、季度和集数信息归档。
- **STRM**：生成文件名和媒体路径时可获得更稳定的媒体元数据。

如果只想修正某个任务的识别结果，优先使用手动绑定；如果是批量命名混乱，优先检查 [[Regex]]、[[Organizer]] 和 [[MediaSettings]] 中的 AI 配置。

---

## 5. API

常用任务接口见 [[API]] 的 **任务管理** 分组。新增维护接口包括：

| 方法 | 路径 | 说明 |
| :--- | :--- | :--- |
| POST | `/api/tasks/:id/clear-cache` | 清理指定任务的转存记录缓存 |
| POST | `/api/tasks/:id/manual-tmdb` | 手动绑定任务的 TMDB 信息 |
