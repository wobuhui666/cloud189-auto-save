# 影巢资源

影巢资源用于按影视条目查询可用网盘资源，并把天翼云盘资源解锁后带入转存任务。当前支持 Cookie 网页解析、OpenAPI OAuth 和 Browser Bridge 网页签名模式。

---

## 1. 功能入口

Web 端新增 **影巢** 页面，常用操作包括：

- 查看影巢配置状态。
- OAuth 授权和撤销授权。
- 使用主项目配置的影巢账号密码调用 Bridge 登录取 Cookie。
- 从 Bridge 同步 Cookie 到主项目。
- 影巢签到。
- 按 TMDB ID 查询电影或剧集的天翼资源。
- 解锁资源并带入创建转存任务。
- 按关键词搜索影巢公开页面结果。

---

## 2. 支持的接入方式

| 模式 | 适用场景 | 说明 |
| :--- | :--- | :--- |
| Cookie 网页解析（可能不可用） | 简单搜索和网页兜底 | 读取影巢页面中可见的天翼链接或资源入口 |
| OpenAPI OAuth | 已有影巢 OpenAPI 凭证 | 用 API Key / Client ID 授权后调用 OpenAPI |
| Browser Bridge | 推荐完整功能（签到不可用） | 使用常驻 Chromium 中的影巢网页签名客户端调用 `/api/customer/*` |

完整的 Bridge Docker 部署方式请参考 [[HDHiveBridge]]。

---

## 3. 配置步骤

进入 **媒体设置 → 影巢资源**：

1. 开启 **启用影巢**。
2. 确认影巢站点地址为 `https://hdhive.com`。
3. 填写影巢网页登录账号和密码。
4. 填写 Browser Bridge 地址和 Token。
5. 开启 **启用 Browser Bridge 签名模式**。
6. 保存配置。

保存后进入 **影巢** 页面：

1. 点击 **账号登录取 Cookie**。
2. 登录成功后点击 **同步 Cookie**。
3. 点击 **签到** 验证 `/api/customer/*` 是否可用。
4. 输入 TMDB ID 查询天翼资源。

账号密码可只保存在主项目配置中；若希望 Bridge 启动自动登录并持续续签登录态（推荐，免去每次手动登录），可在 Bridge 容器额外配置 `HDHIVE_USERNAME` / `HDHIVE_PASSWORD` 环境变量，详见 [[HDHiveBridge]] 的「自动登录与登录态续签」。

---

## 4. 查询资源

查询资源时需要提供：

- 类型：`movie` 或 `tv`
- TMDB ID：例如电影《搏击俱乐部》的 TMDB ID 是 `550`

Bridge 模式下，系统会打开影巢详情页，读取页面渲染出的资源卡片，并优先筛选天翼云盘资源。部分资源需要积分解锁，页面会显示所需积分。

---

## 5. 解锁并转存

点击资源卡片中的 **解锁并转存** 后：

1. 主项目调用 Bridge 的资源解锁接口。
2. Bridge 通过影巢网页签名客户端发起解锁。
3. 解锁成功后解析天翼分享链接和访问码。
4. 主项目自动把分享链接、访问码和资源标题带入创建任务流程。

如果资源已直接暴露天翼链接，则按钮会显示 **直接转存**。

---

## 6. 签到和积分日志

Bridge 模式支持：

- 影巢签到
- 当前用户信息
- 积分日志

这些能力依赖 `BRIDGE_TOKEN` 和有效影巢网页登录态。推荐配置 `DATABASE_URL` 让 Bridge 持久化登录态，避免容器重启后重新登录；在 Bridge 配置账号密码环境变量后，登录态失效时还会自动续签（默认每 25 秒检测一次），无需手动重新登录。

---

## 7. 常见问题

### 7.1 查询不到天翼资源

可能原因：

- 该 TMDB 条目没有天翼云盘资源。
- 当前影巢登录态失效（主项目会在检测到登录失效时自动触发 Bridge 重新登录并重试一次；若 Bridge 配置了账号密码环境变量，通常可自动恢复）。
- Bridge 未部署最新版本或没有启用签名模式。
- 资源只存在 115、夸克、阿里等网盘，不是天翼云盘。

### 7.2 登录后仍提示未配置 Cookie

先在影巢页点击 **同步 Cookie**。如果 Bridge 配置了 `DATABASE_URL`，也可以重启 Bridge 后查看 `/health` 的 `cloudState` 状态。

### 7.3 不想消耗积分怎么办

只使用 **直接转存** 或免费资源。解锁非免费资源前请确认积分消耗。
