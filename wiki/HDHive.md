# 影巢资源

影巢资源用于按影视条目查询可用网盘资源，并把天翼云盘资源解锁后带入转存任务。当前支持 Cookie 网页解析、OpenAPI OAuth 和 Browser Bridge 网页签名模式。

---

## 1. 功能入口

Web 端 **影巢** 页面常用操作包括：

- 查看影巢配置状态。
- OAuth 授权和撤销授权。
- 使用主项目配置的影巢账号密码调用 Bridge 登录取 Cookie。
- 从 Bridge 同步 Cookie 到主项目。
- 影巢签到（手动）与积分日志。
- 按 TMDB ID 查询电影或剧集的天翼资源。
- 解锁资源并带入创建转存任务。
- 按关键词搜索影巢公开页面结果。

完整凭证、Bridge 与**自动签到**配置在 **系统** 页；**媒体** 页仅提供启用开关、站点地址与 Cookie 兜底字段。

---

## 2. 支持的接入方式

| 模式 | 适用场景 | 说明 |
| :--- | :--- | :--- |
| Cookie 网页解析（能力有限） | 简单搜索和网页兜底 | 读取影巢页面中可见的天翼链接或资源入口；仅网页直接暴露分享时才能一键转存 |
| OpenAPI OAuth | 已有影巢 OpenAPI 凭证 | 用 API Key / Client ID 授权后调用 OpenAPI |
| Browser Bridge（推荐） | 签到、积分、解锁、签名接口 | 使用常驻 Chromium 中的影巢网页签名客户端调用 `/api/customer/*`，**支持签到与积分解锁** |

完整的 Bridge Docker 部署方式请参考 [[HDHiveBridge]]。

接入优先级（SDK 内部回退）：OpenAPI OAuth → Browser Bridge 签名模式 → Cookie 网页解析兜底。

## 3. 配置步骤

### 3.1 系统页（完整配置）

进入 **系统 → 影巢资源**：

1. 开启 **启用影巢**。
2. 确认影巢站点地址为 `https://hdhive.com`（或你的镜像站）。
3. 填写影巢网页登录账号和密码（用于 Bridge 登录取 Cookie）。
4. 填写 Browser Bridge 地址和 Token（须与 Bridge 容器 `BRIDGE_TOKEN` 一致）。
5. 开启 **启用 Browser Bridge 签名模式**。
6. 如有 OpenAPI，填写 Client ID / API Key。
7. 按需开启 **每日自动签到**（固定 Cron 或随机时间窗口；可开自动过人机校验）。
8. 保存系统设置。

### 3.2 媒体页（Cookie 兜底）

**媒体 → 影巢设置** 仅维护：

- 启用影巢
- 站点地址
- 网页登录 Cookie

适合只想手填 Cookie、不走 Bridge 的场景。完整 Bridge / 签到请仍到系统页配置。

### 3.3 影巢页验证

保存后进入 **影巢** 页面：

1. 点击 **账号登录取 Cookie**（需 Bridge）。
2. 登录成功后点击 **同步 Cookie**。
3. 点击 **签到** 验证 `/api/customer/*` 是否可用。
4. 输入 TMDB ID 查询天翼资源。

账号密码可只保存在主项目配置中；若希望 Bridge 启动自动登录并持续续签登录态（推荐），可在 Bridge 容器额外配置 `HDHIVE_USERNAME` / `HDHIVE_PASSWORD`，详见 [[HDHiveBridge]]。

---

## 4. 查询资源

查询资源时需要提供：

- 类型：`movie` 或 `tv`
- TMDB ID：例如电影《搏击俱乐部》的 TMDB ID 是 `550`

Bridge 模式下，系统会打开影巢详情页，读取页面渲染出的资源卡片，并优先筛选天翼云盘资源。部分资源需要积分解锁，页面会显示所需积分。

---

## 5. 解锁并转存

点击资源卡片中的 **解锁并转存** 后：

1. 主项目调用 Bridge 的资源解锁接口。
2. Bridge 通过影巢网页签名客户端发起解锁。
3. 解锁成功后解析天翼分享链接和访问码。
4. 主项目自动把分享链接、访问码和资源标题带入创建任务流程。

如果资源已直接暴露天翼链接，则按钮会显示 **直接转存**。

付费或积分未知资源会二次确认，避免误扣积分。

---

## 6. 签到和积分日志

Browser Bridge 模式支持：

- 手动签到（影巢页按钮 / `POST /api/hdhive/checkin` / Bot `/hdhive_checkin`）
- 自动签到（系统页 `hdhive.checkin`，调度任务名「影巢自动签到」）
- 当前用户信息、积分日志

这些能力依赖 `BRIDGE_TOKEN` 和有效影巢网页登录态。推荐配置 `DATABASE_URL` 让 Bridge 持久化登录态；在 Bridge 配置账号密码环境变量后，登录态失效时还会自动续签。

## 7. 常见问题

### 7.1 查询不到天翼资源

可能原因：

- 该 TMDB 条目没有天翼云盘资源。
- 当前影巢登录态失效（主项目会在检测到登录失效时自动触发 Bridge 重新登录并重试一次）。
- Bridge 未部署最新版本或没有启用签名模式。
- 资源只存在 115、夸克、阿里等网盘，不是天翼云盘。

### 7.2 登录后仍提示未配置 Cookie

先在影巢页点击 **同步 Cookie**。如果 Bridge 配置了 `DATABASE_URL`，也可以重启 Bridge 后查看 `/health` 的 `cloudState` 状态。

### 7.3 不想消耗积分怎么办

只使用 **直接转存** 或免费资源。解锁非免费资源前请确认积分消耗。

### 7.4 在媒体页找不到 Bridge 配置

完整 Bridge / 自动签到在 **系统** 页；媒体页只保留 Cookie 兜底，避免两处表单互相覆盖误解。
