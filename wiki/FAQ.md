# 常见问题 (FAQ)

---

## 1. 账号与登录相关

### Q1: 账号登录后不显示容量，或提示接口请求失败

- 请确保天翼云盘账号已实名并成功开启了个盘功能。
- 建议更新至最新版本。
- 尝试使用 **Cookie** 方式登录，比账号密码更稳定。
- 如果开启了二次验证，请先在浏览器登录后获取 Cookie，再通过 Cookie 方式添加账号。

### Q2: 二次验证无法登录

系统暂不支持自动处理短信或 App 二次验证。请先在浏览器登录天翼云盘，获取 Cookie 后通过 **账号** 页 **添加账号** 弹窗的 **Cookie (可选)** 字段添加。

### Q3: 双栈网络环境下登录失败

在 Docker 部署时添加环境变量：

```bash
-e DNS_LOOKUP_IP_VERSION=ipv4
```

## 2. 任务与转存相关

### Q4: 为什么转存任务有延迟？

为了防止被天翼云封禁接口，系统采用了异步队列机制。转存任务通常会在 1-2 分钟内启动，通知也可能有小幅延迟。

### Q5: 支持转存到家庭云吗？

支持。在 **账号** 页 **添加账号** 时，将 **账号类型** 选择为 **家庭云**，并填写 **Family ID**。系统会自动将该账号的任务转存到家庭云目录。

此外，CAS 秒传也支持家庭云中转。在 **媒体** 页 **秒传设置** 中可以配置：

- **启用家庭中转**：秒传时通过家庭云中转，规避个人云风控。
- **优先使用家庭中转**：默认先尝试家庭云秒传，失败再回退个人云。

## 3. 播放与 STRM 相关

### Q6: Emby 提示"无法播放该媒体"

排查步骤：

1. 检查 STRM 文件中的地址是否为正确的外部访问地址。
2. 确保 `8097` 代理端口在 Docker 映射中已打开。
3. 如果未配置 `PUBLIC_BASE_URL`，系统默认使用 `http://127.0.0.1:3000`，远程播放器可能无法访问。建议设置环境变量 `PUBLIC_BASE_URL` 为实际外部地址。
4. 确认 **媒体** 页 **Emby 设置** 中的地址和 API Key 正确。

### Q7: 懒转存的播放速度慢

天翼云盘的转存速度取决于其官方服务器。通常影片的转存需要几秒钟。在转存完成前，播放器可能会出现短暂的缓冲。

## 4. NFO 与刮削相关

### Q8: 刮削后没有生成 NFO 文件

排查步骤：

1. 确认 **媒体** 页 **TMDB 刮削** 已启用。
2. 确认 **TMDB API Key** 已正确填写。
3. 任务需要启用 **启用刮削** 或 **自动整理** 才会触发刮削流程。
4. 检查日志中是否有 TMDB 搜索失败的记录。

## 5. 机器人相关

### Q9: 机器人发送消息后没反应

- 确认服务器能否正常访问 Telegram API（中国大陆服务器通常需要在 **系统** 页 **网络代理** 中设置代理）。
- 确认 **系统** 页 **Telegram 机器人** 中的 **Chat ID** 与您的个人 ID 匹配，机器人不会理睬陌生人的消息。

---

## 6. 其他问题

### Q10: 数据库文件损坏怎么办？

如果遇到数据库错误：

1. 停止容器。
2. 备份映射目录 `data` 下的 `.sqlite` 文件。
3. 删除该 `.sqlite` 文件（注意：这会重置所有设置和账号）。
4. 重新启动容器。

强烈建议定期备份 `data` 目录以防万一。
