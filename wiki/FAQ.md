# 常见问题 (FAQ)

---

## 1. 账号与登录相关

### Q1: 首次打开页面如何登录？有没有默认 admin/admin？

**没有默认 `admin/admin`。** 首次启动若未设置 `ADMIN_PASSWORD`，密码为空，登录页会要求你**自行设置用户名和密码**（至少 6 位）。详见 [[Setup]]。

### Q2: 账号登录后不显示容量，或提示接口请求失败

- 请确保天翼云盘账号已实名并成功开启了个盘功能。
- 建议更新至最新版本。
- 尝试使用 **Cookie** 或 **扫码** 方式登录，比账号密码更稳定。
- 如果开启了二次验证，请先在浏览器登录后获取 Cookie，再通过 Cookie 方式添加账号。

### Q3: 二次验证无法登录

系统暂不支持自动处理短信或 App 二次验证。请先在浏览器登录天翼云盘，获取 Cookie 后通过 **账号** 页 **添加账号** 弹窗的 **Cookie (可选)** 字段添加；或使用 **扫码登录**。

### Q4: 双栈网络环境下登录失败

在 Docker 部署时添加环境变量：

```bash
-e DNS_LOOKUP_IP_VERSION=ipv4
```

## 2. 任务与转存相关

### Q5: 为什么转存任务有延迟？

为了防止被天翼云封禁接口，系统采用了异步队列机制。转存任务通常会在 1-2 分钟内启动，通知也可能有小幅延迟。

### Q6: 支持转存到家庭云吗？

支持。在 **账号** 页 **添加账号** 时，将 **账号类型** 选择为 **家庭云**，并填写 **Family ID**。系统会自动将该账号的任务转存到家庭云目录。

此外，CAS 秒传也支持家庭云中转。在 **媒体** 页 **秒传设置** 中可以配置：

- **启用家庭中转**
- **优先使用家庭中转**

### Q7: 分享内容变了但任务不更新？

在任务菜单执行 **清理缓存**，让系统重新比对分享目录与目标目录。详见 [[Tasks]]。

## 3. 播放与 STRM 相关

### Q8: Emby 提示"无法播放该媒体"

排查步骤：

1. 检查 STRM 文件中的地址是否为正确的外部访问地址。
2. 确保 `8097` 代理端口在 Docker 映射中已打开。
3. 如果未配置 `PUBLIC_BASE_URL` / `system.baseUrl`，系统默认使用 `http://127.0.0.1:3000`，远程播放器可能无法访问。
4. 确认 **媒体** 页 **Emby 设置** 中的地址和 API Key 正确。

### Q9: 懒转存的播放速度慢

天翼云盘的转存速度取决于其官方服务器。通常影片的转存需要几秒钟。在转存完成前，播放器可能会出现短暂的缓冲。

## 4. NFO 与刮削相关

### Q10: 刮削后没有生成 NFO 文件

排查步骤：

1. 确认 **媒体** 页 **TMDB 刮削** 已 **启用刮削**（`tmdb.enableScraper`）。
2. 确认 **TMDB API Key** 已正确填写。
3. 任务需要启用 **启用刮削** 或 **自动整理** 才会触发刮削流程。
4. 检查日志中是否有 TMDB 搜索失败的记录。

## 5. 影巢相关

### Q11: 影巢签到失败或找不到 Bridge 配置

- Bridge / 自动签到完整配置在 **系统** 页，不在媒体页。
- 确认 Browser Bridge 已部署、`BRIDGE_TOKEN` 一致，并已开启签名模式。
- 详见 [[HDHive]]、[[HDHiveBridge]]。

---

## 6. 机器人相关

### Q12: 机器人发送消息后没反应

- 确认服务器能否正常访问 Telegram API（中国大陆服务器通常需要在 **系统** 页 **网络代理** 中设置代理，并勾选 TELEGRAM）。
- 确认 **允许使用的 Chat ID 列表** 或默认 Chat ID 与您的个人 ID 匹配。
- PT 相关命令见 [[Bot]]。

---

## 7. 海报墙 / 文件管理

### Q13: 海报墙榜单订阅不创建任务？

需先在 **系统** 页配置自动追剧默认账号与目录；TMDB/CloudSaver/PT 按你的回退策略配置。详见 [[PosterWall]]。

### Q14: 文件页移动后任务状态异常？

文件管理直接改云盘，不会自动刷新任务缓存；对相关任务执行 **清理缓存**。详见 [[FileManager]]。

---

## 8. 其他问题

### Q15: 数据库文件损坏怎么办？

如果遇到数据库错误：

1. 停止容器。
2. 备份映射目录 `data` 下的 `.sqlite` 文件。
3. 删除该 `.sqlite` 文件（注意：这会重置所有设置和账号）。
4. 重新启动容器。

强烈建议定期备份 `data` 目录以防万一。
