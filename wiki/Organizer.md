# 整理器

整理器用于对已创建任务执行媒体文件整理。它会结合 AI 辅助重命名、媒体库分类命名和 TMDB 信息，将任务中的文件整理成更适合 Emby/Jellyfin/Plex 刮削的结构。

---

## 1. 功能概述

整理器主要解决以下问题：

- 分享资源文件名混乱，不利于媒体服务器识别。
- 剧集、电影、动漫、综艺、纪录片需要分门别类存放。
- 转存完成后希望自动触发重命名、移动、STRM 生成等后续处理。

整理器页本身是任务整理入口，用于查看启用整理的任务和手动执行整理；具体 AI、模板和分类配置在 **媒体** 页完成。

---

## 2. 前置配置

### 2.1 AI 辅助重命名

进入 **媒体** 页，在 **AI 辅助重命名** 中配置：

| 配置项 | 说明 |
| :--- | :--- |
| API 地址 | OpenAI 兼容接口地址 |
| API Key | 接口密钥 |
| 模型名称 | 使用的模型名称 |
| 剧集命名模板 | 剧集整理后的命名格式 |
| 电影命名模板 | 电影整理后的命名格式 |
| AI API 流控 | 限制 AI 调用频率，避免请求过快 |

常见模板示例：

```text
剧集：{name} - {se}{ext}
电影：{name} ({year}){ext}
```

### 2.2 媒体库分类命名

进入 **媒体** 页，在 **媒体库分类命名** 中设置分类目录名称：

| 分类 | 默认用途 |
| :--- | :--- |
| 电视剧 | 剧集资源 |
| 动漫 | 动漫资源 |
| 电影 | 电影资源 |
| 综艺 | 综艺节目 |
| 纪录片 | 纪录片资源 |

分类名称可以按自己的媒体库结构调整。

---

## 3. 在任务中启用整理

创建或编辑任务时，在任务弹窗中勾选 **自动整理**。

常见组合：

| 勾选项 | 作用 |
| :--- | :--- |
| 启用刮削 | 使用 TMDB 获取媒体信息 |
| 懒 STRM | 为懒转存播放场景生成 STRM |
| 自动整理 | 转存完成后执行整理器 |
| 定时任务 | 按 Cron 周期检查任务更新 |

自动追剧和订阅自动创建任务也可以默认启用整理器，具体以创建时的配置为准。

---

## 4. 整理器页使用

进入 **整理器** 页可以看到整理器任务列表。

列表字段包括：

| 字段 | 说明 |
| :--- | :--- |
| 任务名称 | 对应的转存任务 |
| 账号 | 任务使用的账号 |
| 状态 | 是否已启用整理 |
| 最后整理 | 最近一次整理时间 |
| 错误信息 | 最近一次整理失败原因 |

可用操作：

- 使用搜索框 **搜索任务...** 快速过滤任务。
- 点击刷新按钮重新加载列表。
- 对单个任务点击 **执行整理**，立即手动触发整理。

---

## 5. 整理流程

整理器通常按以下顺序处理：

```text
任务文件列表
  ↓
识别媒体类型和剧集/电影信息
  ↓
按命名模板生成目标文件名
  ↓
按媒体库分类确定目标目录
  ↓
执行重命名和归档
  ↓
按任务配置触发 STRM 生成等后续处理
```

实际流程会受任务配置、AI 配置、TMDB 匹配结果和文件状态影响。

---

## 6. API 接口

### 获取整理器任务

```http
GET /api/organizer/tasks
```

### 手动执行整理

```http
POST /api/organizer/tasks/:id/run
```

---

## 使用建议

- 首次配置模板后，先选择少量任务手动点击 **执行整理** 验证效果。
- 文件名非常混乱时，优先保证 AI 配置可用。
- 如果整理失败，先查看整理器列表中的错误信息，再结合实时日志排查。
