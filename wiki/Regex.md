# 正则预设与 AI 辅助重命名

Cloud189 Auto Save 提供两种文件美化方式：正则表达式过滤/替换，以及 AI 辅助重命名。

---

## 1. 正则预设管理

正则预设用于保存常用的过滤和重命名规则，方便在创建任务时一键套用。

### 创建预设

进入 **媒体** 页，在 **正则预设管理** 区域点击 **管理预设** 打开管理弹窗。

每个预设包含以下字段：

| 字段 | 说明 |
| :--- | :--- |
| 名称 | 预设显示名称 |
| 描述 | 可选，说明预设用途 |
| 源正则 (Source Regex) | 文件名匹配正则 |
| 目标正则 (Target Regex) | 替换结果正则 |
| 匹配模式 (Match Pattern) | 文件名匹配条件 |
| 操作符 | `lt`、`gt`、`eq`、`contains`、`notContains` |
| 匹配值 (Match Value) | 用于判断的阈值或关键字 |

**系统** 页的 **正则预设列表** 也可以管理同样的预设数据。

### 在任务中使用

创建或编辑任务时，展开 **高级配置 (正则过滤/替换)** 区域：

1. 在 **正则预设** 下拉框中选择已保存的预设，自动填充下方字段。
2. 也可以手动填写：
   - **源正则 (Source Regex)**：文件名匹配正则
   - **目标正则 (Target Regex)**：替换结果正则
   - **匹配模式 (Match Pattern)**：条件匹配正则
   - **操作符**：`lt` / `gt` / `eq` / `contains` / `notContains`
   - **匹配值 (Match Value)**：判断阈值

## 2. AI 辅助重命名

如果觉得正则太复杂，系统集成了 AI 来自动处理文件名。

### 配置 AI

进入 **媒体** 页，在 **AI 辅助重命名** 中填写：

| 配置项 | 说明 |
| :--- | :--- |
| 启用 | 开关 |
| API 地址 | AI 接口地址，如 `https://api.openai.com/v1` |
| API Key | AI 服务的密钥 |
| 模型名称 | 如 `gpt-3.5-turbo`、`GLM-4-Flash-250414` |
| 剧集命名模板 | 如 `{name} - S{s}E{e}{ext}` |
| 电影命名模板 | 如 `{name} ({year}){ext}` |
| AI API 流控 | 开启后串行排队请求，降低上游并发压力 |

### AI 的作用

- **自动识别集数**：即使文件名混乱，AI 也能识别出是第几集。
- **标准化命名**：按模板格式化为 `SxxExx` 或 `第 xx 集`。
- **过滤冗余信息**：去除字幕组名称、分辨率、水印等。

### 在任务中使用

在创建任务或编辑任务时，勾选 **自动整理** 复选框即可启用整理器。整理器会在转存完成后根据 AI 配置和正则规则对文件进行重命名和归档。

## 3. 常见正则模式参考

| 匹配目标 | 正则表达式 | 说明 |
| :--- | :--- | :--- |
| 集数 (E01) | `E(\d{1,3})` | 匹配 E 开头的数字集数 |
| 分辨率 (4K) | `(4K\|2160p\|1080p)` | 匹配常见分辨率 |
| 年度 (202X) | `(19\|20)\d{2}` | 匹配 4 位数字年份 |
| 后缀 | `\.[^.]+$` | 匹配文件扩展名 |

---

## 使用建议

- 简单过滤优先使用正则，性能最高。
- 复杂重命名推荐开启 AI 模式，能大幅减少配置成本。
- 预设可以在 **媒体** 页和 **系统** 页两处管理，数据互通。
